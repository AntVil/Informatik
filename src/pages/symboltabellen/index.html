Symboltabellen}

Beschreibung}
{0.49\textwidth} Eine Symboltabelle speichert Schlüssel-Wert-Paare. Sie ähnelt in der Funktionsweise einer Abbildung. Mit put wird ein Paar hinzugefügt. Mit get erhält man den Wert für einen Schlüssel.
{0.49\textwidth} \begin{center} \begin{tikzpicture} \fill[cyan, opacity=0.5, rounded corners] (-1.5cm, 0.5cm) rectangle (-0.5cm, 2.5cm); \fill[orange, opacity=0.5, rounded corners] (0.5cm, 0.5cm) rectangle (1.5cm, 2.5cm); \node[node] (node1) at (-1cm, 2cm) {$k_0$}; \node[node] (node2) at (-1cm, 1cm) {$k_1$}; \node[node] (node3) at (1cm, 2cm) {$v_0$}; \node[node] (node4) at (1cm, 1cm) {$v_1$}; \path[->] (node1) edge (node3); \path[->] (node2) edge (node4); \end{tikzpicture} \end{center}

Hashtabelle}
{0.49\textwidth} Eine Hashtabelle ist eine Möglichkeit eine Symboltabelle zu realisieren. Aus einem Schlüssel wird mit einer Hashfunktion ein Index berechnet. Mit dem Index kann man den Wert in ein Array an der Stelle des Index einsetzen.
{0.49\textwidth}
hash: M \rightarrow \mathbb{N}_0
m \mapsto 0
...
\noindent Eine Hashfunktion ist meist nur surjektiv, also kann man für zwei verschiedene Schlüssel den selben Index erhalten. Dies nennt man eine Kollision. \vspace{0.3cm} \noindent
{0.3\textwidth} \textbf{Lineares Sondieren}\\ Bei Kollision werden Werte so lange nach links verschoben, bis eine freie Stelle gefunden wird.
\hspace{0.3cm}
{0.3\textwidth} \textbf{Quadratisches sondieren}\\ Bei Kollision werden Werte in Quadratischen Abständen (links und rechts) vom Index eingefügt.
\hspace{0.3cm}
{0.3\textwidth} \textbf{Überläufer Verkettung}\\ Das Array verwaltet Listen.
